# 🌲🔱 Trident v0.1 Bonsai Review

## Purpose

Review the current Trident architecture before implementation begins.

Goal:

Prune duplication.
Clarify structure.
Protect the trunk.

---

## Review Questions

### Core

- Is this document necessary?
- Does it overlap with another core document?
- Is it doctrine, process, or implementation?
- Should it stay in `core/`?

### Skills

- Does each skill have one responsibility?
- Are any skills duplicates?
- Does every skill belong to a department?
- Should any skill become a playbook instead?

### Playbooks

- Is each playbook operational?
- Does each playbook produce evidence?
- Does each playbook update knowledge?
- Are dependencies clear?

### Intelligence Engine

- Are the seven models clean?
- Are any concepts duplicated?
- Is the flow clear: Entity → Observation → Evidence → Relationship → Confidence → Knowledge → Correlation?

### Repository

- Are folders obvious?
- Is anything misplaced?
- Is anything legacy/reference material mixed with Trident-native material?
- What can be archived or deleted?
