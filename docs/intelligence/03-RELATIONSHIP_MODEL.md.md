# Purpose

Relationships transform isolated entities into intelligence.

An Entity describes what exists.

A Relationship describes how entities connect.

Understanding these connections is the primary objective of intelligence analysis.

---

# Definition

A Relationship is an evidence-supported connection between two or more Entities.

Relationships should never exist without supporting observations.

Relationships may evolve as additional observations are collected.

---

# Philosophy

Entities rarely exist in isolation.

Organizations own infrastructure.

People belong to organizations.

Domains resolve to systems.

Repositories contain secrets.

Applications depend on APIs.

APIs trust identity providers.

Relationships describe these connections.

Knowledge emerges from them.

# Direction

Relationships are directional.

Example:

Repository

↓

HOSTED_ON

↓

GitHub

is different from

GitHub

↓

HOSTS

↓

Repository

Direction should be intentional.

---

# Relationship Lifecycle

Relationships are:

Observed

↓

Validated

↓

Strengthened

↓

Updated

↓

Referenced

↓

Archived

Relationships are never deleted simply because new information is discovered.

Historical intelligence remains valuable.

---

# Relationship Confidence

Confidence belongs to the Relationship.

Not every connection is equally certain.

Confidence should increase as additional observations support the relationship.

---

# Temporal Relationships

Relationships may change over time.

Example:

↓

OWNED_BY

↓

Acme Corp

Observed:
2026

Later:

Repository

↓

OWNED_BY

↓

2028

History should be preserved.

---

# Relationship Evolution

Relationships become stronger through:

New observations

Additional evidence

Independent verification

Repeated occurrence

Mission correlation

Relationships should never become stronger without evidence.

# Design Principles

Relationships should be:

Evidence-backed

Directional

Reusable

Historical

Mission-independent

Relationships create context.

Context creates intelligence.

If two Entities interact,

the interaction deserves investigation.

If the interaction is supported,

it deserves a Relationship.
