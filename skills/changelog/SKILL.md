# 🔱 TRIDENT CHANGELOG SKILL

## Skill Name

`trident-changelog`

## Purpose

Track Trident OS changes, releases, milestones, version history, and development decisions.

## Mission Phase

Primary phase:

- Act

## Responsibilities

This skill is responsible for:

- changelog entries
- release notes
- version summaries
- milestone history
- commit summaries
- development decision logs
- upgrade notes

This skill is not responsible for:

- cyber testing
- mission evidence
- vulnerability reporting
- inventing completed work
- replacing project management

## Inputs

Expected inputs:

- completed changes
- commit history
- version number
- milestone name
- added features
- fixed issues
- known limitations

## Outputs

Expected outputs:

- CHANGELOG.md entries
- release notes
- version summaries
- milestone summaries
- upgrade notes
- project history notes

## Routing Rules

Use this skill when the operator asks to:

- write a changelog
- summarize commits
- prepare release notes
- track versions
- document project history
- name a milestone

Do not use this skill when:

- the request is a cyber mission
- the request is final security reporting
- the task belongs to README writing or project management

## ScopeGuard Requirements

Changelog work does not usually require scope checks.

If a release includes cyber workflows, describe them responsibly and avoid unsafe operational marketing.

## Libraries

Relevant libraries:

This skill may produce:

- changelog entries
- release notes

- version history
- decision summaries

## Mission Handoff

This skill may hand off to:

- `trident-project-manager`
- `trident-readme-writer`
- `trident-repo-architect`

## Limitations

This skill must not:

- claim unreleased work is complete
- exaggerate maturity
- hide breaking changes
- omit known limitations
- confuse roadmap items with shipped features

## Doctrine

Record the journey.

Name the milestones.

Tell the truth.

Ship deliberately.

Discover.

Analyze.

Act.
