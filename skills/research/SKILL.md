# 🔱 TRIDENT RESEARCH SKILL

## Skill Name

`trident-research`

## Purpose

Support structured security research by collecting sources, comparing methods, extracting useful patterns, and turning research into reusable Trident knowledge.

## Mission Phase

Primary phase:

- Discover
- Analyze

## Responsibilities

This skill is responsible for:

- research planning
- source collection
- source evaluation
- methodology comparison
- note synthesis
- reusable insight extraction
- playbook improvement suggestions
- library update recommendations

This skill is not responsible for:

- active testing
- exploitation
- inventing sources
- copying unverified claims
- replacing specialist skills

## Inputs

Expected inputs:

- mission context
- research question
- topic area
- sources
- notes
- existing playbooks
- desired output

## Outputs

Expected outputs:

- research summary
- source notes
- comparison table
- methodology recommendations
- reusable patterns
- playbook update suggestions
- library additions

## Routing Rules

Use this skill when the operator asks to:

- research a topic
- compare techniques
- summarize methodology
- improve playbooks
- study new tools
- turn notes into reusable knowledge

Do not use this skill when:

- the task is active testing
- the task is final reporting
- the task belongs fully to recon, web, API, cloud, or DFIR

## ScopeGuard Requirements

Research should remain educational, defensive, or authorized.

If research turns into operational testing, route through ScopeGuard.

No scope. No strike.

## Mission Handoff

This skill may hand off to:

- `trident-training`
- `trident-analysis`
- `trident-toolsmith`
- `trident-reporting`

## Limitations

This skill must not:

- invent citations
- treat unverified claims as fact
- skip uncertainty labels
- recommend active testing without scope
- replace hands-on validation

## Doctrine

Research feeds methodology.

Methodology feeds missions.

Sources need evaluation.

Knowledge should become reusable.

Discover.

Analyze.

Act.
