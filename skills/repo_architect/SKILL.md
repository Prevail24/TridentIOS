# 🔱 TRIDENT REPO ARCHITECT SKILL

## Skill Name

`trident-repo-architect`

## Purpose

Design, review, and improve the structure of the Trident OS repository so the project remains modular, navigable, and scalable.

## Mission Phase

Primary phase:

- Analyze
- Act

## Responsibilities

This skill is responsible for:

- folder architecture
- naming conventions
- file placement
- module boundaries
- repo cleanup
- documentation structure
- architecture consistency
- expansion planning

This skill is not responsible for:

- cyber operations
- vulnerability testing
- final reporting
- replacing project management
- inventing project requirements

## Inputs

Expected inputs:

- current repo structure
- planned modules
- new files
- architecture goals
- known clutter
- naming conflicts
- future expansion needs

## Outputs

Expected outputs:

- proposed repo structure
- file placement recommendations
- cleanup plan
- naming convention notes
- architecture decision notes
- migration checklist

## Routing Rules

Use this skill when the operator asks to:

- organize folders
- decide where files live
- clean repo structure
- split modules
- rename files
- design architecture
- prevent project clutter

Do not use this skill when:

- the request is a cyber mission
- the task belongs fully to project management
- the operator needs technical testing guidance

## ScopeGuard Requirements

Repo architecture work does not usually require scope checks.

If architecture changes introduce operational cyber workflows, route those workflows through ScopeGuard when used.

This skill may produce:

## Mission Handoff

This skill may hand off to:

- `trident-project-manager`
- `trident-prompt-ops`
- `trident-research`
- `trident-reporting`

## Limitations

This skill must not:

- create unnecessary folders
- over-engineer without purpose
- bury important doctrine
- mix unrelated module responsibilities
- ignore existing project structure

## Doctrine

Architecture creates clarity.

Folders should have purpose.

Modules should have boundaries.

Growth should not create chaos.

Discover.

Analyze.

Act.
